# 提示词示例

## Word / PDF 标准闭环

```text
使用 $taohtml 处理这份 Word 或 PDF。
我已经有内容，但不想自己排版，希望最后得到可以离线使用的 HTML。
请先理解材料并按 TaoHtml 的标准流程开始，不要立刻生成页面。
```

预期流程：材料理解摘要 → 用户确认 → 一次一个关键问题 → 报告设计简报 → 用户确认 → HTML 制作与 QA。

## PDF 转高设计路演课件

```text
使用 $taohtml，把这个 PDF 改成一个高设计感的 HTML 路演课件。
不要丢失重要信息。完整原始页可以通过“原始页”按钮、证据弹窗或附录保留，但主页面要重新设计成适合现场演讲的版本。
使用本地素材，不要依赖外部链接，最后打包成 zip。
```

## 优化已有 HTML 课件

```text
使用 $taohtml，帮我审查并升级这个 HTML 课件。
重点优化故事脊柱、视觉系统、页面节奏、证据处理、翻页器友好的分步展开，以及最终 CTA。
交付前请做资产检查和浏览器 QA。
```

## 复刻 GEO 沙龙案例的工作方式

```text
使用 $taohtml，帮我把这套材料做成一个高设计感 HTML 路演课件。

这不是普通 PPT 美化。请先从观众决策出发，重构演讲主线：
1. 观众现在为什么需要关心这个问题？
2. 哪些真实截图、报告、视频和案例能证明问题正在发生？
3. 需要用哪些机制图、流程图、案例拆解和现场演示页来讲清楚？
4. 最后要如何自然转入产品、服务、报价或下一步行动？

请把真实材料作为证据层处理，主页面负责演讲和说服。
交互必须适合现场翻页器：点击或右键依次展开内容，展开完成后再进入下一页。
使用本地素材、相对路径，检查 1600x900 展示效果，并打包成可迁移 zip。
```

## 忠实迁移 PDF

```text
使用 $taohtml，把这个 PDF 转成一个可离线打开、可翻页、可打包迁移的 HTML 演示稿。
这次优先忠实迁移，不做路演重构。
请保留每一页原始内容，增加键盘翻页、页码、进度条、原始页查看和本地素材文件夹。
```

## 生成口播稿

```text
使用 $taohtml，为这份已经定稿的课件写完整口播稿。
每一页都要包括开场句、主要解释、需要强调的点，以及进入下一页的过渡语。
如果是过渡页，不要写成长篇讲稿，只写自然的承接语。
```

## 设计审查

```text
使用 $taohtml，按照它的高设计评分标准审查这份课件。
告诉我哪些页面显得普通，哪些证据不够强，哪些页面版式重复，以及应该优先重做哪几页。
```
