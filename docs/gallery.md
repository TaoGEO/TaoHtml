# Demo Gallery

这里放 TaoHtml 的真实案例和效果截图。案例优先展示可公开的脱敏页面，不直接公开含有电话、二维码、报价、客户私密材料的完整商业课件。

## GEO 沙龙路演 HTML

这是一套用于线下沙龙的 HTML 路演课件。它从一个 GEO 内容大纲开始，逐步加入真实 AI 问答截图、机制图、复采视频、报告切片、服务转化页和口播稿，最终形成一套可现场演讲、可本地打开、可打包迁移的 HTML Presentation。

### 1. AI 搜索机制页

把复杂的 AI 搜索链路重画成高对比、可演讲的机制页面，保留完整步骤，同时让观众能一眼看到“理解问题、找到证据、判断证据、生成答案、反馈优化”的闭环。

![AI 搜索机制页](assets/cases/geo-salon/01-ai-search-mechanism.png)

### 2. GEO 四个关键词页

把复杂机制压缩成企业能执行的四个抓手：实体、证据、召回、回测。页面采用放射式构图，避免普通卡片堆砌。

![GEO 四个关键词页](assets/cases/geo-salon/02-geo-four-keywords.png)

### 3. AI 检索证据并召回

用一个真实问题演示 AI 不会直接写答案，而是先理解问题、做实体消歧、重写检索词、召回证据、再融合生成。

![AI 检索证据并召回](assets/cases/geo-salon/03-recall-process.png)

### 4. 回测与复采报告

把真实采样录屏和复采报告放进同一套证据页里，说明 GEO 不是“发完内容就结束”，而是需要固定问题、固定平台、保存原始回答和可见信源，持续复查答案变化。

![回测与复采报告](assets/cases/geo-salon/04-retest-report.png)

## 这个案例证明了什么

TaoHtml 不是单页网页模板，也不是普通 PPT 美化。它更适合做这类高要求场景：

- 需要现场演讲，而不是只给人阅读。
- 需要保留真实证据，而不是只讲概念。
- 需要强视觉风格，而不是普通标题加卡片。
- 需要分步展开、视频弹窗、报告滚动、证据切片等 HTML 交互。
- 需要最后打包迁移，在另一台电脑也能打开。

## 推荐使用提示词

```text
使用 $taohtml，把这份材料做成一个高设计感 HTML 路演课件。
不要只做信息排版，要先重构演讲主线：观众是谁、他们需要相信什么、证据怎么出现、最后希望他们采取什么行动。
请把真实截图、报告、视频作为证据层处理，主页面负责演讲和说服。
交互要适合现场翻页器：点击或右键依次展开内容，展开完成后再进入下一页。
使用本地素材和相对路径，最后检查 1600x900 展示效果，并打包成可迁移 zip。
```
