# Report IR v0 对照实验设计简报

## 项目目标

把“案例散落导致证据无法复用”制作成一份可直接现场汇报的 16:9 离线 HTML 演示。

## 受众与行动

- 受众：正在建立案例资产库的咨询或企业服务团队。
- 期望理解：团队缺的不是更多案例，而是让案例变成可调用证据的结构。
- 期望行动：先完成一个最小证据闭环，再扩大案例规模。

## 使用模式

- 现场演讲。
- 5 页。
- 内容按照讲解顺序逐步出现。
- 最终阅读状态必须显示完整内容。
- 完全离线运行。

## 视觉系统

- 主要主题：黑白荧光卡片。
- 需要能在不修改内容的情况下重新编译为严谨咨询报告。

## 固定页面内容

### 第 1 页：核心命题

- 小标题：`01 / 核心命题`
- 主标题第一行：`证据先行，`
- 主标题第二行：`增长才可复制`
- 导语：`把散落的客户案例变成可调用、可验证、可复用的证据系统。`
- 标签：`核心判断`
- 结论：`团队缺的不是更多案例，而是一套让证据随时进入提案、交付与复盘的结构。`

### 第 2 页：当前断点

- 小标题：`02 / 当前断点`
- 标题：`案例很多，证据链仍然断裂`
- 导语：`同一批事实散落在文件夹、聊天记录和个人记忆里，导致价值无法稳定复用。`
- 断点 01：`来源失联`；说明：`截图被复制后失去项目、时间与原始语境。`
- 断点 02：`结论失真`；说明：`销售只记住漂亮结果，却无法说明证据支持了什么。`
- 断点 03：`调用困难`；说明：`新提案开始时，团队仍要从头寻找相似案例。`

### 第 3 页：系统机制

- 小标题：`03 / 系统机制`
- 标题：`四步把案例变成可调用证据`
- 导语：`每一步都生成下一步可直接使用的结构化输入。`
- 01 采集：`保留原始材料、来源、时间与责任人。`
- 02 切片：`拆出问题、动作、结果和适用边界。`
- 03 绑定：`把结论与可追溯证据建立双向链接。`
- 04 调用：`按场景检索并装配到提案、交付和复盘。`

### 第 4 页：固定合成样本

- 小标题：`04 / 固定样本`
- 标题：`同一组数据，直接看复用进度`
- 导语：`这是用于架构对照的合成样本，不代表真实客户数据。`
- 来源标签：`来源：本地合成证据记录；示意 / 待核实`
- 指标 1：`12`；名称：`已归档案例`
- 指标 2：`8`；名称：`已绑定证据`
- 指标 3：`6`；名称：`已进入提案`
- 表头：`阶段 / 数量 / 判断`
- 归档 / 12 / `入口已统一`
- 证据绑定 / 8 / `仍需补齐来源语境`
- 提案调用 / 6 / `优先优化检索与装配`

这些数字只来自 `input/materials/evidence.svg`，只用于合成实验，不能写成真实客户成果。

### 第 5 页：下一步

- 小标题：`05 / 下一步`
- 标题：`先完成最小证据闭环，再扩大案例规模`
- 导语：`用两周把高频场景跑通，让证据从存档进入实际决策。`
- 01 选场景：`锁定一个高频提案或复盘任务。`
- 02 补链路：`为 8 个核心案例补齐来源与结论绑定。`
- 03 测调用：`记录检索时间、采用率与人工修订次数。`
- 标签：`行动结论`
- 结论：`只有进入真实工作流的证据，才是可复用资产。`

## 证据与披露边界

- `input/materials/evidence.svg` 是本地合成 fixture，字节身份可核对，但不是现实世界证据。
- 第 4 页数字必须保留“合成样本、示意或待核实”边界。
- 不新增真实客户、企业、引语、价格、网址、邮箱或电话号码。
- 最终交付说明必须包含简洁的《待核实内容清单》。

## 交付要求

- 入口：`deliverable/index.html`。
- 浏览器双击即可运行。
- 不依赖 CDN、在线字体、远程脚本或远程图片。
- 支持 TaoHtml 当前翻页、动效、阅读最终状态和全屏 Runtime。
