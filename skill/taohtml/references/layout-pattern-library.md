# Layout Pattern Library

Use these patterns to avoid generic title-plus-cards decks. Choose patterns based on slide role, evidence type, and presenter rhythm.

## Selection Rule

Before designing a slide, write:

- Role:
- Audience question:
- Evidence or idea:
- Desired presenter action:
- Pattern:

If the pattern is "cards" three times in a row, change at least one slide.

## Pattern 1: Poster Opener

Use for cover pages, chapter openers, strong propositions, and emotional resets.

Composition:

- Huge headline occupying 35-60% of the slide.
- One short subline.
- One strong visual mass: dark slab, photo, object, source artifact, or geometric cut.
- Minimal body text.

Best for:

- Naming a shift.
- Making a live audience stop reading and start listening.
- Separating chapters.

Avoid:

- Three bullets under the title.
- Decorative icons that weaken the headline.

## Pattern 2: Tension Split

Use for before/after, old/new, problem/solution, search/answer, manual/system.

Composition:

- Two large fields with unequal weight.
- Left side can be problem-heavy; right side can be system or resolution.
- Use contrast in background, scale, or color.
- Add a central hinge: arrow, slash, timeline, or signal line.

Best for:

- Showing why an old method is no longer enough.
- Comparing two operating models.
- Introducing a business shift.

Avoid:

- Equal cards with identical weight when the point is conflict.

## Pattern 3: Evidence Stage

Use when a screenshot, report page, chart, customer message, or source document must carry credibility.

Composition:

- Source artifact is the main object.
- Crop to the decisive area.
- Add 1-3 callouts.
- Place interpretation in a narrow side rail, footer, or conclusion strip.
- Include source button or modal if full context matters.

Best for:

- AI answer examples.
- Product UI.
- Report slices.
- Original PDF pages.
- Legal, medical, finance, government, or B2B proof.

Avoid:

- Making the screenshot too small.
- Rewriting the source into a generic summary.

## Pattern 4: Mechanism Pipeline

Use when explaining a process, causality, model workflow, or operating method.

Composition:

- 4-7 steps maximum on one slide.
- Use directional movement: left-to-right, top-to-bottom, radial, or loop.
- Each step has a verb and a concrete output.
- Add a bottom "why it matters" strip.

Best for:

- How AI retrieves evidence.
- How a report is produced.
- How a service workflow operates.
- How customer behavior becomes a sale.

Avoid:

- Abstract nouns as step titles without actions.
- More than 7 steps with full sentences.

## Pattern 5: System Cutaway

Use when the slide must show layers, infrastructure, workflows, or a production system.

Composition:

- Treat the subject as a machine, factory, stack, or map.
- Use layers, zones, arrows, and labels.
- Show inputs, processing, outputs, and feedback.
- Highlight one layer at a time with staged reveals.

Best for:

- Data factories.
- Knowledge bases.
- Content operations.
- Service systems.
- Enterprise transformation.

Avoid:

- Flat boxes with no relationship.
- A diagram that cannot answer "what moves through this system?"

## Pattern 6: Case Teardown

Use when proving a method through a real example.

Composition:

- Left: case identity and initial question.
- Center: evidence, process, or data.
- Right: diagnosis, result, or implication.
- Use exact names, numbers, screenshots, or source slices.

Best for:

- Client diagnostics.
- AI search sampling.
- Product workflow proof.
- Before/after optimization.

Avoid:

- Generic anonymized cases when the point requires proof.
- Too many lessons on one page.

## Pattern 7: Metric Impact Board

Use when numbers are the main persuasion device.

Composition:

- 1-3 oversized numbers.
- Supporting chart, bar, rank, or comparison.
- Short source note.
- One implication sentence.

Best for:

- Performance improvement.
- Adoption.
- workload reduction.
- risk exposure.
- sampling counts.

Avoid:

- Full tables as the main slide unless the audience needs audit detail.

## Pattern 8: Diagnostic Matrix

Use when the audience needs to locate themselves or compare options.

Composition:

- 2x2 matrix, maturity ladder, risk map, or capability grid.
- Use labels that describe behavior, not vague maturity names.
- Add a highlighted current state and next target.

Best for:

- Service qualification.
- Enterprise readiness.
- tool selection.
- risk diagnosis.

Avoid:

- Matrices with clever labels but no action.

## Pattern 9: Live Demo Shell

Use when embedding or simulating a tool, report, video, or AI answer.

Composition:

- Browser/app/report frame.
- Strong header label that says "simulation", "recording", or "source".
- Large interaction area.
- Presenter-controlled play, replay, zoom, or source modal.
- Footer conclusion strip.

Best for:

- AI Q&A simulations.
- Screen recordings.
- report auto-scroll.
- product walkthroughs.

Avoid:

- Pretending the simulation is a real third-party UI.
- Autoplaying media before the presenter is ready.

## Pattern 10: Offer Stack

Use for service, pricing, implementation plan, or commercial next step.

Composition:

- State the audience's remaining problem first.
- Show 3-5 deliverables as a stack or phased path.
- Include boundary language.
- Include price or CTA only after credibility is established.

Best for:

- Consulting packages.
- service menus.
- product plans.
- conversion pages.

Avoid:

- Starting with price before diagnosis and proof.
- Overpromising outcomes outside the service boundary.

## Pattern 11: Source Appendix Viewer

Use when information must not be lost but is too dense for the main story.

Composition:

- Source page thumbnails or buttons.
- Full-screen modal or appendix slide.
- Optional extracted text panel.
- Optional zoom or scroll.

Best for:

- PDF conversion.
- government or internal reports.
- dense charts.
- audit detail.

Avoid:

- Making appendix material compete with live narrative.

## Pattern 12: Closing Landing

Use to end the presentation with one idea and one action.

Composition:

- Large closing sentence.
- Minimal supporting text.
- Optional QR code, contact, package, or next-step button.
- Strong visual echo of the cover.

Best for:

- Final memory point.
- Sales handoff.
- workshop next action.

Avoid:

- Ending with a dense summary table.

## Rhythm Rules

For a high-design deck, alternate density:

- opener -> evidence -> mechanism -> framework -> case -> system -> offer -> close
- sparse -> dense -> explanatory -> memorable -> proof -> operational -> commercial -> simple

Useful three-slide moves:

- **Scene / Proof / Meaning**: show a behavior, show evidence, explain implication.
- **Problem / Mechanism / Method**: name pain, explain why, give operating model.
- **Source / Teardown / Action**: show artifact, analyze it, turn it into next step.
- **Old / New / System**: contrast, reframe, show repeatable workflow.

## Anti-Patterns

Avoid these unless the user explicitly wants a simple business report:

- Title plus four equal cards on every page.
- Clip-art icons as the main visual.
- Dense paragraph blocks in the main canvas.
- Screenshots used as tiny decorations.
- All pages on the same white background with the same card grid.
- Color palette copied from a reference without copying composition and motion behavior.
- Decorative gradients or blobs with no subject relationship.
- Motion that reveals decoration instead of reasoning.
